Velocity (0.11.9)

Docs
VelocityJS.org

Quickstart

bower install velocity
npm install velocity-animate
<script src="//cdn.jsdelivr.net/jquery.velocity/0.11.8/jquery.velocity.min.js"></script>

Which file should I use?

See VelocityJS.org/#dependencies for more information. jquery.velocity.js will soon be renamed to velocity.js as the jQuery dependency is dropped.

New

  • 0.11.4: AMD and CommonJS module loading without having to shim jQuery.
  • 0.11.1: Huge performance boost, and elements are no longer dirtied with inline styles.
  • 0.11.0: CSS transform setting/getting. VelocityJS.org/#hook
  • 0.10.0: Infinite looping. VelocityJS.org/#loop
  • 0.9.0: Hex colors. VelocityJS.org/#colors
  • 0.8.0: vw/vh CSS units.
  • 0.6.0: display: "auto" and visibility. VelocityJS.org/#displayAndVisibility
  • 0.5.0: Promises. VelocityJS.org/#promises
  • 0.3.0-0.4.0: SVG animation. VelocityJS.org/#svg
  • 0.1.0: stop now stops animations immediately (instead of just clearing the remainder of the animation queue). No other backwards-incompatible changes were made. VelocityJS.org/#stop

Comparisons

  • Famo.us is a full-fledged mobile app framework built around a physics engine.
  • CSS transitions are meant for simple interface flourishes triggered by hover states.
  • jQuery's $.animate() is slow and poorly-equipped for motion design.
  • Velocity allows you to inject rich motion design into your UI.
