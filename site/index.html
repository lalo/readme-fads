About README files

what came first the chicken or the egg?

what came first the popular github repo or the great README file?

A complete and useful README file is the pillar for great open source projects. It enables colaboration and eases the adoption curve of a project. It also helps people use your project with more ease.

In this project we downloaded the top 150 README files of the most starred repos from Github. 150 README files from each language.

Hopefully this project will help people identify the key factors in the README files of the most successfull Github projects. This will enable repos to have a great README from the first commits. Everybody hates when a repo has an empty or a stub README!

Go ahead and select a language from the top bar!

Lessons learned / key insights

  • README files mention the name of the repo a lot (duh!)
  • CSS repos tend to include at least one twitter link in the README
  • Only 4 out of 150 Haskell README files included at least one twitter link
  • Ruby's README files have the biggest word count average
  • On the other hand, Java README files tend to be shorter
  • It's unusual to find a request to donate to the project in a README file
  • VimL README file creators have a guilty conscience, "selfpromotion" was a common word used in titles
  • Demos are popular in CSS repos
  • Contact information is quite popular in objective-c repos
  • Testing information in the README files is common in Javascript and Ruby projects
  • Images are more likely to be found in a Ruby README
  • Perl README files usually don't contain images
